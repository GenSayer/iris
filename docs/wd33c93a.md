# WD33C93A SCSI Bus Interface Controller — Implementer's Reference

This document is a self-contained reference for the Western Digital WD33C93A
SCSI Bus Interface Controller (SBIC), the chip used in SGI Indy / Indigo
workstations behind the HPC3 DMA engine. It is written for someone who needs
to *emulate* the chip. Everything an implementer needs to know about register
layout, command flow, interrupt semantics, and host integration is collected
here.

Sources synthesised:

- WD33C93A November 1990 data sheet and application notes (E031-A, E062-B, etc.)
- MAME `wd33c9x.cpp` / `wd33c9x.h` (BSD-3, most authoritative practical state machine)
- NetBSD and OpenBSD `wd33c93.c` / `wd33c93reg.h` (Wayne Knowles 2001)
- SGI / IRIX kernel driver `wd93.c` / `wd93.h`
- SGI PROM SCSI driver (`scsi.c`, `scsi_common.c`)
- SGI HPC3 glue (`wdsc.c` from NetBSD and OpenBSD)
- The in-progress Rust emulator `src/wd33c93a.rs`

The acronym **SBIC** ("SCSI Bus Interface Controller") is used throughout the
BSD source trees and matches register names like `SBIC_ASR`, `SBIC_CSR`, etc.

## 1. Chip Overview

### 1.1 Physical Interface

The WD33C93A is a 40-pin DIP or 44-pin PLCC, 5V CMOS chip. It bridges:

- An 8-bit host data bus (D7..D0, plus DP for optional odd parity on PLCC).
- An 8-bit SCSI data bus (SD7..SD0, SDP) with built-in 48 mA open-drain drivers.

**Pins (host/processor side):**

| Pin     | Direction | Purpose                                                                                              |
|---------|-----------|------------------------------------------------------------------------------------------------------|
| CLK     | In        | 8–20 MHz square-wave clock. Divisor in OWN ID register (`/2`, `/3`, `/4`).                          |
| MR\     | In        | Master Reset (active-low). 1 µs minimum pulse.                                                       |
| INTRQ   | Out       | Interrupt request to CPU. Cleared by reading the SCSI Status register.                               |
| RE\     | In/Out    | Read enable, qualified by CS\. (Output in WD-bus mode.)                                              |
| WE\     | In/Out    | Write enable, qualified by CS\.                                                                      |
| CS\     | In        | Chip select. Must be inactive during DACK\ or DRQ-driven WD-bus cycles.                              |
| A0      | In        | Single address line. `A0=0` selects ASR/Address Register; `A0=1` selects data port for the indirect register pointed to by AR. |
| ALE     | In        | Direct-addressing latch (multiplexed bus mode). Tie low to use indirect addressing (A0 + AR).        |
| DACK\   | In/Out    | DMA acknowledge (input in DMA/burst); becomes RCS\ output in WD-bus mode.                            |
| DRQ\    | In/Out    | DMA request (output in DMA/burst); becomes DRQ input in WD-bus mode.                                 |
| D7..D0  | I/O       | Host data bus.                                                                                       |
| DP      | I/O       | Optional odd parity on host bus (PLCC only).                                                         |

**Pins (SCSI side, all open-drain):**

| Pin     | Direction (Init / Tgt) | Purpose                                                |
|---------|------------------------|--------------------------------------------------------|
| REQ\    | In (init) / Out (tgt)  | Request a REQ/ACK cycle.                               |
| ACK\    | Out (init) / In (tgt)  | Acknowledge a REQ/ACK cycle.                           |
| ATN\    | Out (init) / In (tgt)  | Attention condition.                                   |
| MSG\    | In (init) / Out (tgt)  | Message phase.                                         |
| C/D\    | In (init) / Out (tgt)  | Control/Data phase.                                    |
| I/O\    | In (init) / Out (tgt)  | Direction of data with respect to the initiator.       |
| BSY\    | I/O                    | Busy. Asserted during arbitration and connection.      |
| SEL\    | I/O                    | Selection.                                             |
| SD7..0  | I/O                    | SCSI data bus.                                         |
| SDP     | I/O                    | SCSI odd parity (PLCC only).                           |

### 1.2 Register Pointer (SASR) and Indirect Addressing

The chip has only **one address line** (A0). All 32 internal registers
(0x00–0x1F) are accessed through a two-step protocol:

- **`A0=0` write:** Stores the data byte into the internal **Address Register**
  (AR / SASR). Only bits 4:0 are kept; the chip's register file is 5 bits wide.
- **`A0=0` read:** Returns the **Auxiliary Status Register (ASR)**.
- **`A0=1` read or write:** Accesses the indirect register currently pointed to
  by AR.

After an `A0=1` access the AR **auto-increments** with three exceptions: it
never auto-increments past the Command (0x18), the Data (0x19), or the
Auxiliary Status (0x1F) registers. The auto-increment lets the host blast the
12-byte CDB or load the three transfer-count bytes back-to-back without
re-writing AR each time.

The chip also supports a **direct addressing** (multiplexed bus) mode where
ALE strobes an address from the data bus. SGI machines do not use this — they
wire A0 to a different physical address and ground ALE, so only indirect
addressing matters for IRIX emulation. In direct mode, ASR maps to register
0x1F and the Address Register read returns ASR.

### 1.3 Chip Variants

| Variant | FIFO | Max sync rate | Notable features                                                  |
|---------|------|---------------|-------------------------------------------------------------------|
| WD33C93 | 5 B  | 2.5 MB/s sync | Original. Cleared all registers on MR. `MR` reset clears Own ID too. |
| WD33C93A| 12 B | 5 MB/s sync   | Adds advanced features (EAF bit), burst DMA, host parity, faster cmd execution, 20 MHz clock support. `MR` hardware reset only resets Own ID (other regs preserved). |
| WD33C93B| 12 B | 10 MB/s (FSS) | Fast SCSI, tagged queueing (Queue Tag register 0x1A), FIFO Full/Empty bit in ASR, target Routine Number bit in Target LUN, Really Advanced Features (RAF). |

For Indy emulation we model the **WD33C93A**. The IRIX kernel detects the
variant by reading the post-reset SCSI Status (0x00 → WD33C93, 0x01 →
WD33C93A) and then probing the WD33C93B-only Queue Tag register (write 0x55,
read back — a 93A returns 0x00 / 0xFF, a 93B returns 0x55).

## 2. Complete Register Reference

The chip exposes 32 indirect registers (0x00–0x1F) plus the auxiliary
status/address register at A0=0.

| 0x01 | R/W | Control                             | Yes                    |
| 0x02 | R/W | Timeout Period                      | Yes                    |
| 0x03 | R/W | Total Sectors / CDB byte 1          | Yes                    |
| 0x04 | R/W | Total Heads / CDB byte 2            | Yes                    |
| 0x05 | R/W | Total Cylinders MSB / CDB byte 3    | Yes                    |

Reading an undefined/unavailable register returns all-ones. Reading the SCSI
Status register **always** clears the INT bit in ASR and lowers the INTRQ pin.

### 2.1 Own ID / CDB Size — 0x00

The value is **sampled and applied only when the `Reset` command (0x00) is
issued**. Writing to it at other times has no effect on chip configuration
(though the value is latched and readable). On the 93A, in "advanced mode"
the same register is overloaded as **CDB Size** during Select-and-Transfer
and Wait-for-Select-and-Receive for unknown command groups.

```
| 7   | 6   | 5     | 4   | 3   | 2 | 1 | 0 |
| FS1 | FS0 |  0    | EHP | EAF |  ID2..ID0 |
   (B: FS1 FS0 RAF EHP EAF  ID2 ID1 ID0)
```

- `ID2..ID0` (0x07): the WD33C93A's own SCSI bus ID (0–7).
- `EAF` (0x08): Enable Advanced Features (93A/93B). Unexpected reselection
  handling, unknown CDB group handling, DPD direction check, status code 0x01
  after Reset.
- `EHP` (0x10): Enable Host Parity checking on D7..D0 + DP (PLCC only).
- bit 5: reserved on 93A (must be 0). On 93B becomes RAF = Really Advanced
  Features (transfer-error detection in target mode, unexpected bus-free
  detection).
- `FS1, FS0` (0xC0): Frequency Select / clock divisor:
  - `00` → divide by 2 → 8–10 MHz input.
  - `01` → divide by 3 → 12–15 MHz input.
  - `10` → divide by 4 → 16–20 MHz input.
  - `11` → undefined.

In the **CDB Size** alias (advanced mode + Select-and-Transfer / Wait-for-
Select-and-Receive with unknown opcode group), bits 0–3 give the total CDB
byte count.

Reset (MR\) default: 0x00.

### 2.2 Control — 0x01

```
| 7   | 6   | 5   | 4   | 3   | 2   | 1  | 0   |
| DM2 | DM1 | DM0 | HHP | EDI | IDI | HA | HSP |
```

- `HSP` (0x01): Halt on SCSI Parity error. Initiator-side parity errors leave
  ACK\ asserted.
- `HA` (0x02): Halt on ATN (target mode). Checks ATN before/every 4096
  bytes/after a transfer.
- `IDI` (0x04): Intermediate Disconnect Interrupt. When set, a valid target
  disconnect during Select-and-Transfer fires status 0x85 and suspends the
  Select-and-Transfer.
- `EDI` (0x08): Ending Disconnect Interrupt. Suppresses the 0x16 completion
  IRQ on Select-and-Transfer until the target physically drops BSY\ — also
  enables Target-mode command chaining.
- `HHP` (0x10): Halt on Host Parity error (93A/93B only). Requires EHP=1.
- `DM2..DM0` (0xE0): DMA Mode select:
  - `000` Polled I/O (default).
  - `001` Burst Mode — DRQ stays asserted while the internal FIFO has
    data/room. (93A/93B only.)
  - `010` WD-Bus / Direct Buffer Access — chip is bus master.
  - `100` DMA Mode — single-byte DMA. DRQ pulsed for each byte.

NetBSD/OpenBSD names: `SBIC_CTL_NO_DMA=0x00`, `SBIC_CTL_HSP=0x01`,
`SBIC_CTL_HA=0x02`, `SBIC_CTL_IDI=0x04`, `SBIC_CTL_EDI=0x08`,
`SBIC_CTL_HHP=0x10`, `SBIC_CTL_BURST_DMA=0x20`, `SBIC_CTL_DBA_DMA=0x40`,
`SBIC_CTL_DMA=0x80` ("single byte dma").

IRIX uses `d_ctrlreg = 0x2D` for burst-mode + EDI + IDI + HSP on 93A/93B,
`d_ctrlreg = 0x8D` for single-byte DMA + EDI + IDI + HSP on the original 93.

Reset (MR\) default: not affected on 93A (preserved). Software RESET command
zeroes it.

### 2.3 Timeout Period — 0x02

8-bit value programming the Select / Reselect timeout. Formula from the
datasheet (msec, input clock in MHz):

    register_value = (Tper * Ficlk) / 80    [round up]

NetBSD/OpenBSD macro:

    SBIC_TIMEOUT(val, clk) = (((val * clk) / 800) + 1)
    /* clk in 0.1 MHz units, val in ms */

A value of 0 disables the timeout altogether. IRIX uses `T93SATN = 32`
(approximately 250 ms at 10 MHz; on 20 MHz machines it doubles to 64) and
`T93SATN_SHORT = 4` (about 30 ms) during hardware inventory.

### 2.4 CDB Bytes 1..12 / Translate-Address Disk Geometry — 0x03..0x0E

These twelve registers serve double duty:

1. **For Select-and-Transfer and Wait-for-Select-and-Receive:** they hold the
   12 bytes of the SCSI Command Descriptor Block.
2. **For Translate Address:** they hold disk geometry and the logical
   address to be translated.
3. **For Send-Status-and-Command-Complete (target mode):** register 11
   (0x0D) holds the status byte to be sent; register 12 (0x0E) holds linked
   command control bits.

| AR   | CDB byte | Translate-Address use            |
|------|----------|----------------------------------|
| 0x03 | 1        | Total Sectors                    |
| 0x04 | 2        | Total Heads                      |
| 0x05 | 3        | Total Cylinders MSB              |
| 0x06 | 4        | Total Cylinders LSB              |
| 0x07 | 5        | Logical Address MSB              |
| 0x08 | 6        | Logical Address byte 2           |
| 0x09 | 7        | Logical Address byte 3           |
| 0x0A | 8        | Logical Address LSB              |
| 0x0B | 9        | (out) Resulting Sector Number    |
| 0x0C | 10       | (out) Resulting Head Number; or Spare Sectors/cyl input for spare comp. |
| 0x0D | 11       | (out) Resulting Cylinder MSB; or Total Sectors/cyl input for spare comp. |
| 0x0E | 12       | (out) Resulting Cylinder LSB                                            |

### 2.5 Target LUN — 0x0F

```
| 7   | 6   | 5    | 4 | 3 | 2   | 1   | 0   |
| TLV | DOK | TRN* | 0 | 0 | TL2 | TL1 | TL0 |   * 93B only (Target Routine Number)
```

- `TL2..TL0` (0x07): LUN (or Target Routine Number on 93B if TRN=1).
- `TRN` (0x20, 93B only): 0 = LUN, 1 = Target Routine Number.
- `DOK` (0x40): Disconnect-OK bit from a received Identify message.
- `TLV` (0x80): Target LUN Valid — an Identify message was received.

After Select-and-Transfer completes, this register also holds the **Target
Status byte** received from the target.

### 2.6 Command Phase — 0x10

7-bit field (mask 0x7F). Tracks progress through the Select-and-Transfer,
Reselect-and-Transfer, and Wait-for-Select-and-Receive sequences. The host
can both **read** it on completion/abort to know how far the command got, and
**write** it before re-issuing the same combination command to **resume** at
a specific phase.

Common values (Select-and-Transfer):

| Value | Meaning                                                        |
|-------|----------------------------------------------------------------|
| 0x00  | No target selected. Disconnected.                              |
| 0x10  | Target selected (now Connected as Initiator).                  |
| 0x20  | Identify message sent.                                         |
| 0x30  | Command phase started, no bytes transferred yet.               |
| 0x3x  | x bytes transferred (0x36 = 6, 0x3A = 10, 0x3C = 12).          |
| 0x41  | Save Data Pointer message received.                            |
| 0x42  | Disconnect message received (bus not yet free).                |
| 0x43  | Target disconnected (bus free) after Disconnect message.       |
| 0x44  | Target reselected us.                                          |
| 0x45  | Identify message received from target (LUN matches).           |
| 0x46  | Data Transfer phase complete (TC reached zero).                |
| 0x47  | Status phase started.                                          |
| 0x50  | Status byte received, stored in Target LUN.                    |
| 0x60  | Command Complete message received.                             |
| 0x61  | Linked Command Complete message received (93A/93B).            |

### 2.7 Synchronous Transfer — 0x11

```
| 7   | 6   | 5   | 4   | 3   | 2   | 1   | 0   |
| FSS*| TP2 | TP1 | TP0 | OF3 | OF2 | OF1 | OF0 |
                                       * FSS = Fast SCSI Sel, 93B only with 16-20 MHz clock.
```

- `OF3..OF0` (0x0F): synchronous REQ/ACK offset, 0..12 (values 13–15 clamp to
  12). **Offset 0 = asynchronous** for data phases. The maximum offset is
  bounded by the FIFO depth: 4 on the 93 (5 actually but only 4 should be
  used), 12 on the 93A/93B.
- `TP2..TP0` (0x70): transfer period in internal-clock cycles:
  - `000` → 8 cycles (also default)
  - `010` → 2, `011` → 3, `100` → 4, `101` → 5, `110` → 6, `111` → 7
  - Cycle = `divisor / (2 * Finput_MHz)` µs.
- `FSS` (0x80, 93B only): Fast SCSI doubles frequency.

Asynchronous transfer happens whenever **offset is 0** OR the bus phase is
not a data phase. For other phases the chip always transfers asynchronously.

**WD33C93 vs WD33C93A:** the 93 required programming `desired + 1` in the
period field; the 93A takes the desired value directly. The 93's
recommended max offset is 4 even though the FIFO can hold 5. The 93A's max
useful offset is 12.

IRIX `transval()` returns:
- period > 175 → 0 (use async, slowest synchronous is 800 ns)
- period < 50 (Fast SCSI, 93B only) → FSS bit | 2..4
- otherwise → (period + 24) / 25

### 2.8 Transfer Count MSB / Middle / LSB — 0x12..0x14

24-bit transfer counter. Loaded before a `Send`, `Receive`, `Transfer Info`,
or any combination command. Decrements on every byte transferred over the
SCSI bus. **Reaches zero → successful-completion interrupt.**

Special cases:

- Loading the register with zero, or setting the SBT (Single Byte Transfer)
  bit in the Command register, disables counting; the command completes after
  one byte.
- After a successful transfer the register is always zero, **even in SBT
  mode** on the 93A (this was a 93 quirk that the 93A removed).
- On halt-on-error / phase change / abort, the register holds the number of
  bytes that were **not** transferred. The internal FIFO may still hold some
  bytes that have made it into the chip but not yet onto the SCSI bus, so
  this count is the authoritative measure of what reached the SCSI bus.
- Bus-master DMA host transfers (WD-bus) can leave a discrepancy because
  bytes can be in flight on the host side too.

`SBIC_TC_MAX = (1 << 24) - 1`. IRIX clamps DMA chunks to 0xFFFFFC to avoid
fencepost issues at the 16 MB boundary.

### 2.9 Destination ID — 0x15

```
| 7   | 6   | 5  | 4 | 3 | 2   | 1   | 0   |
| SCC | DPD | DF | 0 | 0 | DI2 | DI1 | DI0 |
   (B: SCC DPD TG1 TG0 0 DI2 DI1 DI0)
```

- `DI2..DI0` (0x07): SCSI ID of the device to select/reselect.
- `DF` (0x20, 93A E/F only): disable the DPD direction check feature and the
  link from Send-Status-and-Command-Complete to Wait-for-Select-and-Transfer.
- `DPD` (0x40, advanced mode only): expected data-phase direction.
  - 0 = Data Out (host → target)
  - 1 = Data In (target → host)
  - The chip raises an "unexpected info phase" interrupt during
    Select-and-Transfer if the target's actual data phase doesn't match.
- `SCC` (0x80): Select Command Chain — used only when issuing
  Reselect-and-Transfer with `EDI = 1`. 0 = chain to Send-Status-and-Command-
  Complete; 1 = chain to Send-Disconnect-Message.

NetBSD/OpenBSD aliases: `SBIC_SID_DPD = 0x40` (also called `SBIC_SID_FROM_SCSI`
when used as direction indicator), `SBIC_SID_TO_SCSI = 0x00`, `SBIC_SID_SCC =
0x80`, `SBIC_SID_IDMASK = 0x07`.

IRIX names this "WD93DESTID" and uses bit 6 (`SCDMA_IN = 0x40`) ORed with the
target ID to tell the 93A the data direction during Select-and-Transfer.

### 2.10 Source ID — 0x16

```
| 7  | 6  | 5   | 4 | 3   | 2   | 1   | 0   |
| ER | ES | DSP | 0 | SIV | SI2 | SI1 | SI0 |
```

- `SI2..SI0` (0x07): SCSI ID of the device that selected/reselected the chip
  (valid only when SIV=1).
- `SIV` (0x08): Source ID Valid. Set when the other device asserted its own
  bus ID bit during the (re)selection in addition to ours; if only our ID was
  asserted, SIV=0.
- `DSP` (0x20): Disable Select Parity — accept selections/reselections with
  parity errors.
- `ES` (0x40): Enable Selection — allow target-mode selection.
- `ER` (0x80): Enable Reselection — allow initiator-mode reselection.

After a hardware reset (MR\) `ES`, `ER`, and `DSP` are forced to 0; bits 0–3
are *preserved* across MR\ but cleared by the software RESET command. IRIX
writes 0x80 (ER set, ES cleared) on every reset because we never act as a
target.

### 2.11 SCSI Status — 0x17 (read-only)

Read-only. Contents are frozen between the moment INTRQ asserts and the
moment the host reads this register; reading it clears INT in ASR and lowers
INTRQ. See §5 for the full status-code table.

```
| 7   | 6   | 5   | 4   | 3   | 2   | 1   | 0   |
| SS7 | SS6 | SS5 | SS4 | SS3 | SS2 | SS1 | SS0 |
```

The upper nibble defines the interrupt **group**:

| SS7..SS4 | Group                                                              |
|----------|--------------------------------------------------------------------|
| `0000`   | Reset state.                                                       |
| `0001`   | Command completed successfully.                                    |
| `0010`   | Command paused or aborted.                                         |
| `0100`   | Command terminated due to error.                                   |
| `1000`   | SCSI bus event requires service.                                   |

For status codes in groups `0001`, `0010`, `0100`, and `1000`, when the low
three bits represent an information phase, the encoding is the **MCI**
(MSG, C/D, I/O) value:

| MCI | Phase             |
|-----|-------------------|
| 000 | Data Out          |
| 001 | Data In           |
| 010 | Command           |
| 011 | Status            |
| 100 | Unspecified Out   |
| 101 | Unspecified In    |
| 110 | Message Out       |
| 111 | Message In        |

### 2.12 Command — 0x18

8-bit write triggers execution. The host must:

- Not write the Command register when ASR.CIP or ASR.INT are set.
- Not write a Level II command when ASR.BSY is set.
- Wait approximately 7 µs after reading SCSI Status before issuing a new
  command (otherwise the chip may set LCI and ignore the write).

```
| 7   | 6..0           |
| SBT | Command code   |
```

- `SBT` (0x80): Single Byte Transfer qualifier. When set in conjunction with
  an information-transfer command (`Transfer Info`, `Send`, `Receive`),
  disables the transfer counter and forces exactly one byte to move.

### 2.13 Data — 0x19

PIO data port. Host reads/writes this register when ASR.DBR is set and the
chip is *not* in DMA mode (DACK\ inactive). Also accessible during the
advanced-mode unexpected-reselection sequence so the host can fetch the
Identify message.

DMA / Burst / WD-Bus modes route the FIFO to the DMA pins instead, but the
Data register address is what DACK\ implicitly selects in those modes — the
chip ignores the AR contents during DMA cycles.

### 2.14 Queue Tag — 0x1A (WD33C93B only)

Holds the second byte (Tag value) of tagged queue messages on the 93B.
Sample-write 0x55 / read-back is used to discriminate 93A from 93B; on a 93A
this register doesn't latch — most parts return 0x00 or 0xFF.

### 2.15 Auxiliary Status — A0=0 read, or AR=0x1F (read-only)

```
| 7   | 6   | 5   | 4   | 3    | 2 | 1  | 0   |
| INT | LCI | BSY | CIP | FFE* | 0 | PE | DBR |
                          * 93B only
```

See §3 for full semantics.

ASR can be read at any time **except during a DMA bus cycle** (DACK\ low in
DMA/Burst, or DRQ high in WD-Bus mode).

## 3. Auxiliary Status Register Semantics

### 3.1 INT — 0x80 ("Interrupt Pending")

Set whenever the chip has put a fresh code into the SCSI Status register and
is asserting INTRQ. **Cleared** by reading the SCSI Status register (AR=0x17,
A0=1 read). Once INT is cleared, INTRQ deasserts.

In a real WD33C93A only one interrupt is presented at a time. MAME and the
in-progress Rust implementation use an internal "IRQ FIFO" of pending status
codes so that command flows that semantically push two consecutive statuses
(e.g. SELECT_ATN → SELECT_SUCCESS then REQ_SEND_MESG_OUT) get sequenced
correctly: after the host reads SCSI Status and clears INT, the next entry
is loaded into SCSI Status and INT is re-raised before INTRQ ever physically
glitches low. Implementers should follow that model.

### 3.2 LCI — 0x40 ("Last Command Ignored")

Set when a command write hit the chip too soon (within ~7 µs of the last
SCSI-Status read, or while CIP/INT were still set, or while BSY=1 for a
Level II command). The command is **silently dropped**. LCI is cleared the
next time a command is successfully accepted.

IRIX's `wd93cmd_lci()` reads ASR after every `putcmd93` and re-issues the
command in a loop while LCI is set. Drivers that don't expect to lose
commands will hang if you set LCI gratuitously.

### 3.3 BSY — 0x20 ("Busy / Level II in progress")

Set while a Level II command is executing. Only the Command register (when
CIP=0), the Data register, and ASR are guaranteed to be accessible. Cleared
when the Level II command finishes.

### 3.4 CIP — 0x10 ("Command In Progress")

The internal microcontroller is interpreting the Command register; the
Command register itself is unavailable. Cleared as soon as the chip
recognises which command was issued. MAME notes that CIP being set "only
means the chip is interpreting the contents of the Command Register, it
shouldn't actually be set" — in practice MAME never sets it. The IRIS
emulator sets it briefly to make IRIX's `WAIT_CIP` poll terminate cleanly.

### 3.5 FFE — 0x04 (WD33C93B only)

FIFO Full/Empty indicator. When writing to the FIFO, FFE=1 means FIFO
empty; when reading, FFE=1 means FIFO full. Lets the host blast up to 12
bytes in/out without polling DBR per byte. On a 93A this bit reads as zero.

### 3.6 PE — 0x02 ("Parity Error")

A parity error was detected on a data byte. SCSI parity is checked on bytes
received from the SCSI bus during writes to the bus; host parity (EHP=1) is
checked on bytes from the host bus on writes to the SCSI bus.

PE is set *regardless* of HHP/HSP — those bits control whether the chip
halts the transfer, not whether the error is reported. PE is cleared by the
next command.

For synchronous transfers parity is checked every 4096 bytes (or at end of
transfer); for asynchronous, every byte.

### 3.7 DBR — 0x01 ("Data Buffer Ready")

The PIO heartbeat:

- During a **send** (host → SCSI): DBR=1 means the chip is ready to take
  another byte; DBR=0 after the host writes one. Goes to 1 again once the
  byte made it onto the SCSI bus.
- During a **receive** (SCSI → host): DBR=1 means a byte is waiting in the
  Data register; goes to 0 when the host reads it.
- DBR is also set whenever a Send / Receive / Transfer-Info command is
  issued (so the host can preload the FIFO immediately).
- DBR is *always* zero when the chip is in DMA mode — bytes go through the
  DMA pins.

The exception (added by MAME comment) is that in advanced mode a reselected
93B may need the host to read the Identify message from the Data register
without DBR being set first.

The high bit (`0x80` = SBT) is a qualifier, not a command — it sets
single-byte mode on information-transfer commands.

### 4.1 Reset (0x00)

Software reset. Aborts whatever is in flight, samples the Own ID register
(latches clock divisor, advanced features, host parity, SCSI ID), and clears
internal state.

Result:
- DBR ← 0; INT ← 1; INTRQ asserted.
- All SCSI bus signals released.
- FIFO, transfer counter, offset/period state machines cleared.
- Registers 0x01–0x16 zeroed. Command register (0x18) zeroed.
- SCSI Status set to **0x00** if EAF=0 in Own ID, or **0x01** if EAF=1.

Hardware reset (MR\) differs slightly on the 93A: it only zeros the Own ID
register, leaving 0x01–0x15 and the lower nibble of Source ID intact.

### 4.2 Abort (0x01)

Disconnected state: stops an in-flight Select / Select-and-Transfer /
Reselect / Reselect-and-Transfer. If we already won arbitration, drops the
ID bits while SEL\ is asserted, watches for BSY\ for 200 µs. No response →
go bus-free, fire "paused/aborted" interrupt (group 0x2_). Response → fire
"successful completion" instead.

Connected state (T or I): stops a Send / Receive / Transfer Info /
Transfer Pad / Select-and-Transfer data-phase. The host **must continue
servicing DBR** (for Receive) or **must NOT service DBR** (for Send) until
the chip raises the abort interrupt — this is how the FIFO drains.

After abort: Transfer Count holds bytes-not-transferred; chip stays
Connected (in T or I), ready for the next command including a resume.

On the WD33C93 the post-abort interrupt was 0x23; on the WD33C93A it's
0x13 (during a Select-and-Transfer), 0x22 if aborting from Select/Reselect,
0x25 if aborting during selection-in-progress.

### 4.3 Disconnect (0x04)

Releases all SCSI signals and (in target mode) returns the bus to Bus Free.
If issued during an active Level II command, the Level II terminates
immediately.

### 4.4 Assert ATN (0x02) — initiator only

Asserts ATN\. The chip automatically negates ATN\:

- Before the last byte of a Transfer Info in response to Message Out.
- When the Identify Out is sent during Select-and-Transfer.
- When the bus goes Bus Free.

The two `Select-with-ATN*` commands set ATN automatically before releasing
SEL.

### 4.5 Negate ACK (0x03) — initiator only

Drops ACK after a Message In Transfer Info (the chip leaves ACK asserted so
the host can choose to reject the message by issuing Assert ATN before
Negate ACK), or after halting on a parity error.

For accepting a message, just issue Negate ACK alone.
For rejecting a message, issue Assert ATN then Negate ACK.
For "initiator detected error", issue Assert ATN then Negate ACK.

### 4.6 Select-with-ATN / Select-without-ATN (0x06 / 0x07)

Initiator-side selection. Pre-load Destination ID with the target. The chip
arbitrates, asserts SEL\, drives the OR-ed IDs, releases BSY\, waits up to
the Timeout Period for the target to assert BSY\. Successful selection
fires status **0x11** (SELECT complete). For `Select-with-ATN` the chip
asserts ATN\ before releasing SEL\.

A reselection by another device during arbitration aborts the command and
fires a `0x8x` service-required interrupt instead.

### 4.7 Reselect (0x05) — target mode

Like Select-without-ATN but asserts I/O\ at the end of arbitration. The
WD33C93A becomes Connected-as-Target on success.

### 4.8 Select-with-ATN-and-Transfer / Select-without-ATN-and-Transfer (0x08 / 0x09)

The big "do everything" Level II commands. The chip handles the entire
SCSI operation:

1. Arbitrate and select the target.
2. If ATN was asserted, send the Identify message during the first Message
   Out phase. The Identify byte is generated as
   `0x80 | (ER ? 0x40 : 0) | (Target_LUN & 0x07)`.
3. Send the CDB (6/10/12 bytes selected by group code or by Own ID in
   advanced mode).
4. Optionally transfer data (length = Transfer Count register).
5. Receive the Status byte (stored in Target LUN register).
6. Receive the Command Complete message.
7. Fire status 0x16 (or hold it until bus-free if EDI=1).

The Command Phase register increments through the values listed in §2.6.
Any unexpected phase triggers a "terminated" interrupt and leaves the
Command Phase register pointing to the last completed step.

These commands can be **resumed** by writing the Command Phase register
and re-issuing the same command. The chip walks the resume table:

| Phase | Resume action |
|-------|---------------|
| 0x10  | After successful selection. |
| 0x20  | After Identify out; expect command phase (implied Negate ACK). |
| 0x30  | After REQ for command. |
| 0x41  | After save data pointer; expect data/status/msg in (implied Negate ACK). |
| 0x42  | After disconnect msg (implied Negate ACK). |
| 0x44  | After reselect by the target. |
| 0x45  | More data in data-phase (implied Negate ACK). |
| 0x46  | After data done, expecting status or save-DP/disconnect. No implied Negate ACK. |
| 0x50  | After status, complete status phase (implied Negate ACK). |
| 0x60  | Complete command complete (implied Negate ACK). |

### 4.9 Receive (0x10–0x13) / Send (0x14–0x17) — target mode

Target-side counterparts to Transfer Info. They set the MCI signals
according to the table in §1 and pump the FIFO. The host either polls DBR
or uses DMA. Termination conditions: transfer count → 0, single byte done
(SBT=1), parity error (halt-enabled), ATN seen with HA=1, Abort issued,
Disconnect issued.

### 4.10 Transfer Info (0x20) — initiator

Pumps bytes for the currently asserted SCSI bus phase. The host sets up
Transfer Count first, optionally sets SBT in the Command write for
single-byte mode, and either polls DBR or uses DMA.

Successful completion: TC reaches zero. The chip waits for the next REQ\
from the target before firing the "successful completion" interrupt
(SCSI Status `0x18 | MCI`). The exception is Message-In: it interrupts
immediately, with **ACK\ left asserted** so the host can examine the
message byte and decide whether to issue Negate ACK or Assert ATN +
Negate ACK.

A parity error (SCSI side HSP=1, or host side HHP=1) terminates the
command. SCSI parity errors leave ACK\ asserted to also halt the target.

A target disconnect (BSY\ dropped) or a phase change before TC=0 also
terminates the command and fires a "terminated" interrupt.

### 4.11 Transfer Pad (0x19) — initiator (93/93B; **not on 93A**)

Pumped junk bytes to flush an unexpected residue. Removed from the 93A's
command set per Application Note E062-B. NetBSD/OpenBSD still define
`SBIC_CMD_XFER_PAD = 0x19` for older parts.

### 4.12 Translate Address (0x18)

Logical → physical disk address translation. Inputs:
- Total Sectors register (CDB byte 1).
- Total Heads register (CDB byte 2).
- Total Cylinders register (CDB bytes 3–4).
- Logical Address register (CDB bytes 5–8).
- (Optional) Head Number register = spare sectors/cyl (0 = no
  compensation), Cylinder Number register = total sectors/cyl when spare
  comp is active.

Outputs:
- Sector Number (CDB byte 9).
- Head Number (CDB byte 10).
- Cylinder Number (CDB bytes 11–12).

Status 0x15 on success, 0x45 ("logical address too large") if cylinder >=
total_cylinders or a division overflows.

MAME implements this in a single block under `COMMAND_CC_TRANSLATE_ADDRESS`
and IRIX never invokes it (Indy doesn't use this feature), but the chip
provides it for systems that do CHS-based geometry.

### 4.13 Set IDI (0x0F)

Sets the Control register's IDI bit while a Level II command is busy (i.e.
when the register file is otherwise inaccessible). Lets the host enable
intermediate disconnect interrupts during an already-running
Select-and-Transfer for overlapped I/O.

### 4.14 Send-Status-and-Command-Complete (0x0D) / Send-Disconnect-Message (0x0E) — target mode, 93A/93B only

`Send-Status-and-Command-Complete` sends the status byte from CDB byte 11
followed by a Command Complete (0x00), Linked Command Complete (0x0A), or
Linked Command Complete with Flag (0x0B) message based on CDB byte 12 bits
0 and 1.

`Send-Disconnect-Message` sends a Disconnect message and disconnects.
When IDI=1 it sends a Save Data Pointers (0x02) message first.

Both can be chained from Reselect-and-Transfer when `EDI=1`, with the chain
target chosen by the SCC bit in Destination ID.

### 4.15 SBT (0x80) — Single Byte Transfer qualifier

ORed with any information-transfer command (Send, Receive, Transfer Info).
Forces exactly one byte to transfer regardless of Transfer Count. After
the transfer, Transfer Count register is corrupted on the 93A and must be
reloaded (the 93 preserved it as an undocumented quirk).

States: D = Disconnected, T = Connected-as-Target, I = Connected-as-Initiator.

## 6. Interrupt Model

### 6.1 INTRQ pin

INTRQ is asserted (high or low according to how the host wires it; the chip
itself drives an open-drain pin) whenever **ASR.INT = 1**. The pin
deasserts when ASR.INT clears.

The data sheet does **not** specify a particular polarity — the WD33C93A
INTRQ pin is described as an output that the host's interrupt controller
samples however it wires it. In SGI/IRIS the line is wired to the HPC3
interrupt source and routed as `INT3` (the IRIS HPC3 driver treats it as
edge-triggered through the HPC3 register layer; the chip itself is
level-driven).

### 6.2 Clearing INT

Reading the SCSI Status register (AR=0x17, A0=1 read) **always**:

1. Returns the current status code.
2. Clears ASR.INT.
3. Drops INTRQ.

On the IRIS chip, if more than one status is queued internally (the IRQ
FIFO model), reading SCSI Status loads the next one into the SCSI Status
register and re-asserts INT *immediately* without dropping INTRQ — the
host sees a fresh interrupt with the new status. This matches MAME's
behaviour and matches what IRIX expects when sync negotiation produces
back-to-back interrupts.

### 6.3 LCI semantics

LCI is the chip's way of saying "I couldn't accept your command write".
Causes:

- Command written within ~7 µs of the previous SCSI Status read.
- Command written while INT=1 or CIP=1.
- Level II command written while BSY=1.

LCI is cleared the next time a command write succeeds. Drivers handle this
by either polling-and-retrying (IRIX `wd93cmd_lci`) or by reading SCSI
Status before re-issuing the command.

### 6.4 PE semantics

Set unconditionally on a parity error, even when HHP/HSP are 0 (the halt
bits control whether the *transfer* aborts, not whether the error is
reported). PE is cleared by the next command.

### 6.5 BSY / CIP

- `CIP=1`: only ASR is readable. The chip is interpreting the Command
  register.
- `BSY=1`: ASR, Command (when CIP=0), and Data register are accessible.
  All other registers may give stale or transient values.

In practice the chip rarely sets CIP for any visible period (MAME comments
to that effect). Setting BSY for the duration of Level II commands is
correct, however.

## 7. Bus State Machine

The chip is always in one of three modes:

- **Disconnected** (`MODE_D`): no SCSI bus connection.
- **Connected as Target** (`MODE_T`): we are the target.
- **Connected as Initiator** (`MODE_I`): we are the initiator.

Within each mode the chip walks the SCSI protocol phases:

### 7.1 BUS FREE → ARBITRATION → SELECTION

On a `Select*` command from Disconnected:

1. Wait for the bus to be free (`BSY` and `SEL` both negated for the bus-free
   delay).
2. Drive our own ID bit on SD7..SD0; assert `BSY\`.
3. After arbitration delay, examine the bus. If `SEL\` is asserted by
   someone, lose arbitration — restart wait.
4. Find the highest asserted ID bit; if it's ours, win — assert `SEL\`.
5. Drive (our_ID | target_ID) on SD bus.
6. Release `BSY\`. Assert `ATN\` (Select-with-ATN only).
7. Wait for target to assert `BSY\`. Selection complete.

MAME's `ARB_WAIT_BUS_FREE → ARB_CHECK_FREE → ARB_EXAMINE_BUS →
ARB_ASSERT_SEL → ARB_SET_DEST → ARB_RELEASE_BUSY → ARB_DESKEW_WAIT` chain
is a faithful breakdown. Time the timeout from `ARB_TIMEOUT_BUSY`.

### 7.2 RESELECTION (target reconnect, initiator side)

A previously disconnected target asserts `SEL\` and `I/O\` with both its
own ID and ours on the SD bus. The chip:

- In normal mode: fires 0x80 ("Reselected") immediately so the host can do
  protocol manually.
- In advanced mode with ER=1: continues into Message In to fetch the
  Identify message, then fires 0x81 (idle) or 0x27 (during
  Select-and-Transfer) with the Identify byte in the Data register or
  Target LUN, ACK\ left asserted.

The Source ID register holds the reselecting target's ID with SIV=1.

### 7.3 COMMAND phase

`C/D\` asserted, `MSG\` negated, `I/O\` negated. Target asserts REQ and
the chip pumps CDB bytes from the FIFO (Select-and-Transfer) or from the
host via Transfer Info.

### 7.4 DATA IN / DATA OUT phase

`C/D\` negated. Direction by `I/O\`. The chip can use synchronous transfer
(Synchronous Transfer register offset ≠ 0) or asynchronous.

### 7.5 STATUS phase

`C/D\` asserted, `MSG\` negated, `I/O\` asserted. One byte from target to
initiator. Select-and-Transfer auto-stashes it in Target LUN.

### 7.6 MESSAGE IN / MESSAGE OUT

`C/D\` and `MSG\` both asserted.

Message In behaviour is special: when a Transfer Info completes a Message
In, the chip leaves **ACK\ asserted** so the host can examine the message
byte (in the Data register or FIFO) and respond:

- Accept → issue Negate ACK.
- Reject → issue Assert ATN, then Negate ACK (target will request Message
  Out for "Message Reject").
- Initiator Detected Error → same.

Status 0x20 ("paused Msg In") indicates this state.

### 7.7 DISCONNECT / RECONNECT

Normal disconnect sequence (initiator view):

1. Target sends Save Data Pointers message (0x02). Chip fires 0x21 during
   Select-and-Transfer. Host updates DMA pointers. With Set IDI / IDI=1
   the chip also delivers an 0x85 "disconnect" interrupt.
2. Target sends Disconnect message (0x04). Chip advances Command Phase to
   0x42.
3. Target drops BSY\. Bus free. Chip advances Command Phase to 0x43.
   - If IDI=1, fires 0x85 to suspend Select-and-Transfer.
   - Otherwise sits idle waiting for the target to reconnect.

Reconnect: target reselects us; if its ID matches Destination ID and the
Identify LUN matches Target LUN, the chip silently advances Command Phase
to 0x44/0x45 and continues the Select-and-Transfer. If anything mismatches
the chip fires a terminated interrupt.

In **advanced mode** the chip handles unexpected reselections more
gracefully — see §5.5 status 0x27 / 0x81.

## 8. DMA Operation

### 8.1 DRQ / DACK handshake

In DMA modes (`DM2..DM0 = 001` Burst, `100` Single-byte DMA):

- The chip asserts **DRQ\** (active low, open-drain — needs a host
  pull-up) when the FIFO has a byte to deliver (read) or room to take a
  byte (write).
- The host's DMA controller responds with **DACK\** + RE\ or WE\.
- In Single-byte DMA the chip pulses DRQ\ once per byte.
- In Burst DMA the chip leaves DRQ\ asserted while the FIFO has work to
  do; the DMA controller can run as long as DRQ\ remains low.

CS\ must be **inactive** during a DACK\ cycle; DACK\ implicitly selects
the Data register regardless of the AR contents.

DRQ/DACK timing (from data sheet):

- Burst Write: tDLWL = 0 ns DACK low → WE low; tWLQH = 75 ns max WE low →
  DRQ high (deassert); WE pulse width 50 ns min.
- Burst Read: tDLRL = 0 ns DACK low → RE low; tRLQH = 75 ns max RE low →
  DRQ high; RE pulse 80 ns min.

In **WD-Bus / Direct Buffer Access mode** (`DM2..DM0 = 010`) the roles
reverse: DRQ becomes an *input* from the host saying "you may transfer";
DACK becomes the **RCS\** output asserted as a chip-select for the
external buffer. RE and WE become outputs the chip drives. This mode is
intended for embedded designs and is not used on SGI.

### 8.2 Transfer Count

Before issuing any Send / Receive / Transfer Info / combination command
that transfers more than one byte, the host loads the 24-bit Transfer
Count register. The chip decrements on every byte that hits the SCSI bus.
When TC reaches zero:

- For Transfer Info (non-Msg-In): wait for next REQ, then fire the
  "successful completion" interrupt with MCI = new phase.
- For Send/Receive: fire "successful completion" (0x13 / 0x14 depending
  on ATN).
- For Select-and-Transfer Data Phase: advance Command Phase to 0x46
  silently and proceed to next phase.

If TC starts at zero or SBT=1, the chip transfers exactly one byte and
completes.

On abort/halt/phase-change the TC register holds bytes-not-transferred.
The internal FIFO may hold some bytes too (see §2.8).

### 8.3 DMA vs PIO selection

The mode is selected by the Control register's DM bits. Polled I/O
(DM=0) → host polls DBR and accesses the Data register. Any DMA mode →
host accesses the FIFO via DACK/DRQ; **DBR is always 0 in DMA mode**.

NetBSD `wd33c93_dmaok()` decides per-transfer whether to DMA; when DMA is
chosen, the driver sets `Control |= SBIC_CTL_BURST_DMA` for SGI HPC3 and
issues `SBIC_CMD_XFER_INFO`.

### 8.4 Halting DMA

There is no explicit "halt DMA" bit. To halt:

1. Stop the external DMA controller (HPC3 in IRIS).
2. Wait for ASR.BSY and ASR.CIP to drop, polling ASR.
3. Read the residual count out of Transfer Count.
4. Either let the chip finish naturally on phase change, or issue Abort.

NetBSD's `wd33c93_dma_stop` does exactly that: spin on `(asr &
(SBIC_ASR_BSY|SBIC_ASR_CIP))`, then call the platform's `sc_dmastop`
hook, read residual via `SBIC_TC_GET`, then reset Control register to a
no-DMA value (`SBIC_CTL_EDI | SBIC_CTL_IDI`).

### 8.5 IRIX / HPC3 interaction

The HPC3 (High Performance Controller 3) is the SGI Indy DMA engine. It
sits between the WD33C93A and main memory. Per channel it maintains a
scatter-gather descriptor list and a small FIFO.

Key behaviours the emulator needs to match:

- HPC3 drives DACK\ and RE\/WE\ in response to the chip's DRQ\.
- IRIX programs the WD33C93A with Burst-mode DMA (Control = 0x2D = `DM=001
  | EDI | IDI | HSP`) on the 93A/93B.
- HPC3 XIE / EOX (End-Of-Transfer) bits fire when a descriptor completes;
  the IRIX driver re-arms the descriptor chain mid-SCSI-transfer for long
  reads/writes. This is why the in-progress Rust emulator pauses at
  256 KB boundaries and raises 0x48/0x49 ("unexpected info phase") so
  IRIX's `unex_info()` re-arms.
- HPC3 SCSI control register at offset `scsi0_ctl` includes a chip-reset
  bit (`scsi_dmactl_reset`) that wdsc_match toggles during probe to drive
  the WD's MR\ pin.

The HPC3-side counter (visible to IRIX) may diverge from the WD33C93A's
internal Transfer Count by the size of the internal 12-byte FIFO. The
WD's count is authoritative for SCSI-bus byte count; the HPC3 count is
authoritative for memory bytes touched.

## 9. Synchronous Transfer

### 9.1 Sync Transfer Register format

Already documented above. To summarise:

- **OF3..OF0**: REQ/ACK offset 0..12 (clamped). 0 = async for data phases.
- **TP2..TP0**: period in internal-clock cycles.
- **FSS** (93B only): doubles the frequency.

### 9.2 Sync negotiation (SDTR)

The chip itself does *not* automatically negotiate. The host driver runs
the SDTR (Synchronous Data Transfer Request) exchange via Transfer Info
commands:

1. After winning Select-with-ATN, the chip presents a Message Out phase.
2. Driver sends `[Identify, Ext Msg Header (0x01), Ext Msg Length (0x03),
   Sync Code (0x01), Period (period in 4ns units), Offset (REQ/ACK
   offset)]`.
3. Target responds with Message In: its own SDTR with target's preferred
   period and offset.
4. Driver computes the min of the two periods and the min of the two
   offsets, programs the Sync Transfer register, and proceeds with the
   command. If the target rejects sync, leave the Sync Transfer register
   at 0 (asynchronous).

IRIX `_sync_setup()` shows the full dance, including handling of
target-initiated sync, rejection paths, and the dreaded WD bug where the
chip hangs waiting for more REQs if you switch to async without resetting.

### 9.3 Choosing sync vs async

- Offset = 0 in Sync Transfer register → always asynchronous.
- Non-data phases (Command, Status, Message) are always asynchronous
  regardless of register value.
- Data phases with offset > 0 are synchronous.

IRIX's recommended max offset is **10 for the 93A** (better throughput
than 12 on IOC2 according to internal Application Note JL0014), **4 for
the 93** (FIFO is only 5 deep and offset=5 had a documented bug).

## 10. Initialization Sequence

Combining the IRIX `wd93reset()`, NetBSD `wd33c93_reset()`, and SGI PROM
`scsi.c`:

```c
/* 1. Tell the host adapter to drive MR\ for ~1 µs (HPC3 scsi_dmactl_reset). */
write_hpc3_scsi_ctl(haa->scsi_dmactl_reset);
delay(1 us);
write_hpc3_scsi_ctl(0);

/* 2. Load Own ID with desired SCSI ID, clock divisor, EAF/RAF/EHP. */
my_id = host_scsi_id | SBIC_ID_FS_16_20 | SBIC_ID_EAF /* | SBIC_ID_RAF */;
SET_SBIC_myid(sc, my_id);

/* 3. Issue Reset command. */
SET_SBIC_cmd(sc, SBIC_CMD_RESET);

/* 4. Wait for INT (ASR.INT=1, INTRQ asserted). */
SBIC_WAIT(sc, SBIC_ASR_INT, 0);

/* 5. Read SCSI Status — clears INT, returns 0x00 (93) or 0x01 (93A/B). */
GET_SBIC_csr(sc, csr);

/* 6. Probe variant. CDB_1 (0x03) holds the microcode revision when RAF=1
 *    on a 93B.  Write 0x55 to Queue Tag (0x1A); 93B keeps it, 93A returns 0. */
GET_SBIC_cdb1(sc, rev);
if (csr == SBIC_CSR_RESET_AM) {
    SET_SBIC_queue_tag(sc, 0x55);
    GET_SBIC_queue_tag(sc, val);
    chip = (val == 0x55) ? WD33C93B : WD33C93A;
}

/* 7. Set Control register for DMA mode + halt-on-parity + disconnect ints. */
SET_SBIC_control(sc, SBIC_CTL_EDI | SBIC_CTL_IDI /* | SBIC_CTL_BURST_DMA */);

/* 8. Set Timeout register. */
SET_SBIC_timeo(sc, SBIC_TIMEOUT(250, clock_in_hundredkHz));

/* 9. Enable reselection. */
SET_SBIC_rselid(sc, SBIC_RID_ER);

/* 10. Async sync register. */
SET_SBIC_syn(sc, 0);
```

NetBSD additionally issues `SBIC_CMD_ABORT` before Reset to clear any
stale state ("hardware-reset followed by software-reset" idiom).

## 11. Typical Command Flow (Initiator Mode)

Walkthrough of a SCSI READ(6) via the chip in non-combination mode
(individual commands; this is the path IRIX takes for non-DMA transfers
and the path the IRIS Rust emulator currently uses for PIO data-in):

### 11.1 Setup

```c
SET_SBIC_destid(sc, target_id);            /* DESTINATION_ID */
SET_SBIC_tlun(sc, lun);                    /* TARGET_LUN */
SET_SBIC_syn(sc, sync_period_offset);      /* SYNC_TRANSFER */
SBIC_TC_PUT(sc, 0);                        /* TRANSFER_COUNT = 0 for select */
SET_SBIC_cmd(sc, SBIC_CMD_SEL_ATN);        /* COMMAND = 0x06 */
```

The chip arbitrates, selects, and asserts ATN.

### 11.2 SELECT_SUCCESS

ASR.INT=1. Host reads SCSI Status → **0x11** ("Select complete"). The
chip is now Connected-as-Initiator. ASR.INT clears; if the chip has a
second status queued (REQ for Msg Out) it immediately re-raises with
**0x8E** (REQ Msg Out).

### 11.3 IDENTIFY out

Host loads Transfer Count = 1, FIFO with the Identify byte
`(0x80 | (disconnect_ok ? 0x40 : 0) | lun)`, and issues
`SBIC_CMD_XFER_INFO` (0x20). DBR-based PIO delivers the byte, ATN is
auto-negated, then the chip fires **0x18 | 0x02 = 0x1A** ("Transfer
complete; new phase Command").

### 11.4 CDB out

Host loads Transfer Count = 6, then writes the 6 CDB bytes to the Data
register one at a time as DBR cycles. Then **0x18 | 0x01 = 0x19**
("Transfer complete; new phase Data In") fires.

### 11.5 DATA IN

Host pre-programs the HPC3 DMA channel for the read, sets Control to
Burst DMA mode, loads Transfer Count to the byte count, and issues
`SBIC_CMD_XFER_INFO`. The chip pumps bytes via DRQ/DACK; HPC3 writes to
memory. When TC reaches 0 the chip fires **0x18 | 0x03 = 0x1B**
("Transfer complete; new phase Status").

### 11.6 STATUS in

Host issues `SBIC_CMD_XFER_INFO` with SBT (or TC=1). Chip reads the
status byte; pushes it via DBR. The chip then fires **0x18 | 0x07 = 0x1F**
("Transfer complete; new phase Msg In").

### 11.7 MESSAGE IN

Host issues `SBIC_CMD_XFER_INFO` with SBT. Chip reads the Command
Complete message byte (0x00). At this point **ACK\ is left asserted** and
the chip fires **0x20** ("Msg In paused with ACK").

### 11.8 NEGATE ACK → BUS FREE

Host issues `SBIC_CMD_NEGATE_ACK` (0x03). Target drops BSY\. The chip
fires **0x85** ("Disconnect"). Bus is free.

### 11.9 Combination-command short cut

If the host instead issues `SBIC_CMD_SEL_ATN_TRANSFER` (0x08) after
loading the CDB into registers 0x03..0x0E, the chip walks the entire
sequence above autonomously. Only one interrupt is delivered (0x16) at
the end, with the target status byte in Target LUN (0x0F). This is what
IRIX uses for high-throughput I/O.

## 12. Error Handling

### 12.1 Selection / reselection timeout — 0x42

Target didn't assert BSY\ within the Timeout Period. The chip fires 0x42,
returns to Disconnected. IRIX retries `T93SRETRY = 3` times before giving
up.

### 12.2 Unexpected disconnect — 0x41

Target dropped BSY\ before the expected sequence finished. The chip
returns to Disconnected. IRIX simply disconnects and reports an error
upwards.

### 12.3 Unexpected phase — 0x48 | MCI

Target requested a phase the chip wasn't expecting (e.g. status before TC
hit zero, or wrong phase during Select-and-Transfer). The chip transitions
to Connected-as-Initiator and leaves the bus in the requested phase, so
the host can use Transfer Info to handle the situation manually.

IRIX `unex_info()` handles common variants:

- `ST_UNEX_SSTATUS` during a data phase with leftover TC → assume the
  target finished early (Inquiry / Mode Sense), set TC to 0, resume
  Select-and-Transfer at PH_DATA.
- `ST_UNEX_SMESGIN` with message byte = 1 → extended message in,
  probably target-initiated sync negotiation. Drive the full SDTR
  exchange via Transfer Info.
- `ST_UNEX_SMESGIN` with message = 2 (Save Data Pointers) → ack the
  message, resume from PH_SAVEDP.
- `ST_UNEX_SMESGIN` with message = 3 (Restore Pointers) → ack, undo any
  tentative pointer save, resume from PH_IDENTRECV.
- `ST_UNEX_CMDPH` at phase = PH_SELECT → some devices skip the message
  phase; set PH to 0x20 and resume.
- `ST_UNEX_CMDPH` at phase = PH_CDB_6 with cmd length > 6 → the chip
  doesn't know about command groups 2/3/4/6/7; the host manually
  transfers the remaining CDB bytes via Transfer Info then resumes.

### 12.4 Parity errors — 0x43 / 0x44

A SCSI-bus or host-bus parity error during a halt-enabled transfer.
0x43 = ATN not asserted, 0x44 = ATN asserted. Transfer Count holds
unsent bytes. For initiator SCSI parity errors, ACK\ is left asserted so
the target halts; host can choose to send a "Message Reject" or
"Initiator Detected Error" via Assert ATN + Negate ACK.

### 12.5 Invalid command — 0x40

The Command register was loaded with a value invalid for the current
state. The chip doesn't change state.

### 12.6 Reset / external bus reset — 0x00 / 0x01

Triggered by MR\, the Reset command, or another device asserting SCSI
RST\. Host should reinitialise everything.

## 13. IRIX-Specific Notes

### 13.1 `wd93intr()` flow (from `wd93.c`)

```c
void wd93intr(int adap) {
    wd93dev_t *dp = &wd93dev[adap];
    nested_spinlock(&dp->d_qlock);
    /* On Everest only: turn off DMA right away. */
    if (getauxstat() & AUX_INT)
        handle_intr(dp);
    nested_spinunlock(&dp->d_qlock);
}
```

`handle_intr` is the heart. It:

1. Spins until ASR.CIP and ASR.BSY are both clear (with R4400 the chip
   sometimes leaves these set briefly).
2. Reads ASR. If `LCI` is set, restarts the last command. If `PE` is
   set, marks the request as `SC_PARITY`.
3. Reads, in this order: `Target LUN (0x0F)`, `Command Phase (0x10 via
   auto-incr)`, `Source ID (0x16)`, `SCSI Status (0x17)`. Reads them
   *before* SCSI Status because the auto-increment then lines up with
   the natural SBIC layout.
4. Dispatches on SCSI Status via a chain of `if/else if`. The most
   common cases (DISC, SAVEDP, SATOK, RESELECT) come first.

### 13.2 Register usage during ISR

The ISR reads (in order): ASR, TARGET_LUN (0x0F), COMMAND_PHASE (0x10),
SOURCE_ID (0x16), SCSI_STATUS (0x17). It never reads the Data register
unless it must consume an Identify byte during a 93A reselect (status
0x81 or 0x27).

### 13.3 Command Phase usage

IRIX leans heavily on the Command Phase register. After every
Select-and-Transfer completion or abort it inspects Command Phase to
decide what state the bus was left in and how to resume. Common values
it specifically checks:

- `PH_NOSELECT` (0x00) — selection failed.
- `PH_SELECT` (0x10) — target selected but command not even started.
- `PH_IDENTSEND` (0x20) — Identify message sent.
- `PH_CDB_6` (0x36) — first 6 CDB bytes sent; common point for the
  "unknown command group" workaround.
- `PH_SAVEDP` (0x41) — save data pointer received.
- `PH_DISCRECV` (0x42) — disconnect message received.
- `PH_DISCONNECT` (0x43) — target physically disconnected.
- `PH_RESELECT` (0x44) — target reconnected.
- `PH_IDENTRECV` (0x45) — Identify received from target.
- `PH_DATA` (0x46) — data transfer complete.
- `PH_STATUSRECV` (0x50) — status received.
- `PH_COMPLETE` (0x60) — command complete msg received.

To resume a Select-and-Transfer, IRIX writes Command Phase, sets up
Destination ID + LUN + Sync + maybe Transfer Count, and re-issues
`C93SELATNTR` (0x08). The chip checks Command Phase to know where to
restart.

### 13.4 Phase mismatch handling

IRIX uses `wd93_transfer(dp, PH)` to resume a Select-and-Transfer at a
specific Command Phase value. The common idioms:

- After parity error: `wd93_transfer(dp, PH_IDENTRECV)` to continue.
- After Save Data Pointers: `wd93_transfer(dp, PH_SAVEDP)`.
- After unexpected Disconnect message handled via unex_info: same.
- After 93A "incorrect data" status 0x47 quirk:
  `wd93_transfer(dp, phase)` (a 93A bug — just resume).

### 13.5 Reselect / reconnect

The 93A path (status 0x81 "ST_93A_RESEL"):
1. Read Source ID — has SIV=1 and target ID.
2. Read Data register — has the Identify message.
3. Find the matching channel; call `wd93continue(dp, sp, PH_IDENTRECV)`.

The 93 path (status 0x80 "ST_RESELECT"):
1. Source ID has target ID (no SIV).
2. Write Command Phase = PH_RESELECT.
3. Wait for next interrupt with status 0x8F (ST_REQ_SMESGIN).
4. Read Data register for the Identify byte.
5. Call `wd93continue(dp, sp, PH_RESELECT)`.

### 13.6 The `setdest()` / `wd93_transfer()` idiom

```c
putreg93(WD93DESTID, sp->wd_dstid);  /* target ID | (read ? 0x40 : 0) */
putreg93(WD93LUN, sp->sr_lun);
putcmd93(C93SELATNTR);               /* re-issue 0x08 */
```

The "DPD" bit (0x40 in Destination ID) is set when the data direction is
*from* SCSI (i.e. a SCSI read). IRIX uses this to make Select-and-Transfer
verify the target's data phase matches our expectation. Without advanced
mode the bit is ignored.

## 14. SGI / HPC3 Integration

### 14.1 Address mapping

On both NetBSD and OpenBSD wdsc.c, the WD33C93A is mapped at:

- `WD_ASR` = `ha_devoff + 3` (or `ha_devoff + 0 + 3` on hpc1 with NetBSD
  byte-laning).
- `WD_DATA` = `ha_devoff + 4 + 3` (or `ha_devoff + 3 + 4`).

The `+3` accounts for the HPC3's 32-bit register width: the chip is wired
to the low byte of the 32-bit bus, so byte accesses must hit offset 3
(big-endian MIPS). The 4-byte stride between ASR and DATA is the HPC3's
register-spacing convention.

In summary: from the host's view, the chip occupies two 32-bit
locations 4 bytes apart, each containing the chip's 8-bit register in
its low byte. The high 24 bits are don't-care on reads, ignored on
writes.

NetBSD `wd33c93reg.h` even has the SGI-specific note:

> We access them via separate handles because some people *cough*SGI*cough*
> like to keep them apart.

### 14.2 HPC3 DMA wiring

The HPC3 SCSI channel hooks into the WD33C93A via:

- **DRQ\** → HPC3 SCSI DMA request input.
- **DACK\** → HPC3 SCSI DMA acknowledge output.
- **INTRQ** → HPC3 SCSI INT3 source.
- **MR\** → controllable via `scsi0_ctl` register's reset bit.

The HPC3 SCSI channel has a small FIFO and a scatter-gather descriptor
chain in main memory. Descriptors carry `(buffer_addr, byte_count,
flags)`. Flags include EOX (end of chain), XIE (interrupt on completion
of this descriptor).

### 14.3 Control register configuration on SGI

For 93A/93B, IRIX uses `d_ctrlreg = 0x2D`:

- bit 0 HSP=1 (halt on SCSI parity)
- bit 2 IDI=1 (intermediate disconnect interrupt)
- bit 3 EDI=1 (ending disconnect interrupt)
- bit 5 DM=001 (burst DMA mode)

For the original 93, IRIX uses `d_ctrlreg = 0x8D`:

- bit 0 HSP=1
- bit 2 IDI=1
- bit 3 EDI=1
- bit 7 DM=100 (single-byte DMA mode)

### 14.4 Reset behaviour

The HPC3 has its own reset bit for the WD33C93A on the SCSI control
register (`scsi_dmactl_reset`). Writing it asserts MR\; clearing it
releases. NetBSD's `wdsc_match()` uses this for probe: assert reset,
delay 1 ms, release, then read ASR. A live WD33C93A returns `SBIC_ASR_INT
= 0x80` (the post-reset INT bit) when read.

### 14.5 Interrupt acknowledgement

The IRIS implementation collapses two interrupt acks into the SCSI Status
read:

1. Reading SCSI Status (AR=0x17) clears the chip's INT and lowers INTRQ.
2. The IRIS `cb.clear_pdma_int()` hook also drops the HPC3 PDMA SCSI
   completion bit — because both halves of the SCSI line share INT3 and
   the kernel only acks once at the chip level.

Without that coupling the PDMA bit stays asserted forever after the chip
INT is cleared, IP2 storms, and the IRIX 6.5 miniroot stalls.

## 15. Implementation Cheat Sheet

For an emulator, the minimum viable behaviour is:

1. Implement the 32-register file with auto-increment-except-COMMAND/DATA/
   ASR semantics.
2. Implement ASR with INT/LCI/BSY/CIP/PE/DBR bits (FFE only if claiming
   93B compatibility).
3. Implement an internal IRQ FIFO that drives ASR.INT and the SCSI Status
   register; pop on SCSI Status read.
4. Implement the Reset command properly (sample Own ID, push status
   0x00 or 0x01).
5. Implement Select-with-ATN (0x06) and Select-with-ATN-and-Transfer
   (0x08); these are the two IRIX uses for initiator I/O.
6. Implement Transfer Info (0x20) with DBR-based PIO and DRQ/DACK-based
   DMA.
7. Implement Negate ACK (0x03) and Abort (0x01) so the driver can recover.
8. Implement enough Disconnect / status sequencing that
   Select-and-Transfer can return all relevant Command Phase values.
9. Implement Translate Address (0x18) if you target firmware that uses
   it (IRIX doesn't).
10. Implement Reselect-path statuses (0x81, 0x27) only if you support
    multi-target overlapped I/O.

The most subtle correctness requirements are:

- Reading SCSI Status **must** clear INT and **may** immediately re-raise
  it with the next queued status. No INTRQ glitch must occur for queued
  back-to-back interrupts.
- Transfer Count register must hold bytes-not-transferred after abort or
  phase change.
- DBR semantics differ between Send (set when chip wants more) and
  Receive (set when byte ready). Always 0 in DMA mode.
- Command Phase register increments and resume semantics must match
  the data sheet table exactly — IRIX writes it back to control resume.
- LCI must trip if the host writes Command while ASR.INT, ASR.CIP, or
  ASR.BSY are set (but BSY only blocks Level II commands).
- ATN auto-negation behaviour (before last byte of MSG OUT, etc.) must
  not be skipped — drivers rely on it.
